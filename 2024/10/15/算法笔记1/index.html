算法学习笔记1：排序+二分的思考

快速排序



  快排的思想不难，简单来说就是分而治之，代码则通过递归进行即可，但是我在写代码过程中遇到了很神奇的边界问题，就是在对子序列进一步排序时，这个排序的区间是到 i 还是到 i - 1 或者 j 还是 j - 1。
  幸运的是在题目的评论区遇到了大佬的解答，真是应了雷总的一句话“这个世界上百分之九十的问题都已经有答案了”，
  解答如下
  首先需要明白一点，就是快排代码中的 x 只体现一个中间值的作用，而不是中间位置。
  然后在代码循环要结束的时候，j - i = 1，那么再运行一次的话变成了 i - j = 1，此时[l, j]区间 为小于 x 的数，[i, r]区间 为大于 x 的数。那么这么去看好像下一段分治区间选择[l,j] 或者 [l, i]都可以，但事实并不是这样。
  解答如上图所示，会陷入死循环，可以拿 l = 0, r = 1 模拟一下，假如此时 mid = l + r >> 1, quick_sort(q, l ,i )运行完结果会出现两种可能.

  1. 结果：i = 1, j = 0, 此时会陷入死循环，所以不能使用 i 作为边界，必须使用 i - 1。
  2. 结果：i = 0, j = 0, 此时进入下一次分治就变成了 [0, -1]，明显越界，对于此边界问题，改变 mid 的值即可，即令 mid = l + r + 1 >> 1。

归并排序

  归并排序的思想感觉和快排是反过来的，快排是不断划分将大序列拆小，使其有序，但归并是从小序列合并到大序列。
  在归并排序中也遇到了类似的边界问题，这个是边界的同性问题我觉得，就是在合并子序列的过程中选择了[l, mid - 1]，那么就会出现类似的越界问题。
  例如当l = 0, r = 1, 那么 mid = l + r >> 1 = 0，那么 mid - 1 = -1，明显越界。所以要使用 mid - 1则需l + r + 1 >> 1，如果使用 mid 作为下一段合并区间则不用。

循环不变量

  很惊艳的一个概念（取自代码随想录），二分经常会遇到边界问题如何选择是选择不等于还是等于，傻傻分不清。
  循环不变量就是指二分循环的过程中始终保持同类型的区间，例如都是左闭右闭或者都是左开右闭，这个在二分中是循环不变的，所以就知道什么时候需要 +1 -1了

数的范围


  这道题是二分的经典题型，它的关键点在于找到左右两个边界，那么在二分时就需要注意，例如在找左边界时，当num[mid] == x,此时是需要缩小右边界的，所以r = mid - 1,同理即可写出右边界代码。

二分查找

搜索插入位置


  这道题的临界位置也很有意思，最后返回的插入位置该如何选择，left 还是 right 还是 right + 1。
  分析方法很简单，就是考虑最后一步的情况，最后一步应该是left = right了，那么mid = left，如果nums[mid] > target，则right = mid - 1,那么插入位置应该是 right + 1的位置，如果nums[mid] < target，那么left = mid + 1，即为需要插入的位置。
  所以最终的答案返回left 或者 right + 1 即可，单独考虑边界情况划拉一下就懂了。
